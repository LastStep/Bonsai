## Slash Commands

Seven workflows are curated as **slash commands**, meaning they can be invoked directly from the Claude Code prompt using `/<name>` syntax. These are registered as Claude Code skills (via `.claude/skills/` files) during workspace generation.

The curated slash command workflows are:

- `/planning` ‚Äî Start end-to-end planning for a new feature
- `/code-review` ‚Äî Review agent output against the plan
- `/pr-review` ‚Äî Review a pull request for correctness and standards
- `/security-audit` ‚Äî Run a security audit on the codebase
- `/issue-to-implementation` ‚Äî Take an issue from intake through to shipped code
- `/test-plan` ‚Äî Design a structured test plan for a feature
- `/plan-execution` ‚Äî Execute an assigned plan step by step

<Aside type="tip">
Not every workflow is a slash command. The curated set is intentionally small to keep context usage low ‚Äî each slash command consumes context budget at session start. Workflows like `session-logging` and `reporting` are loaded on demand.
</Aside>

## Workflows by Role

### Orchestration (Tech Lead)

- **planning** ‚Äî Translates requirements into a structured implementation plan with tiers, phases, and verification steps.
- **issue-to-implementation** ‚Äî The full autonomous workflow: intake an issue, analyze it, plan, dispatch to code agents, review, iterate, and close.
- **code-review** ‚Äî Reviews agent output against the plan for correctness, standards, and security compliance.
- **pr-review** ‚Äî Reviews a pull request: scope, correctness, security, performance, test coverage.
- **test-plan** ‚Äî Designs test coverage strategy: scope, priorities, and allocation across test types.

### Execution (Code Agents)

- **plan-execution** ‚Äî Reads an assigned plan and executes it step by step, reporting progress.
- **api-development** ‚Äî Spec-first API workflow: define the contract, generate scaffolding, implement, test, and document.
- **reporting** ‚Äî Generates a structured completion report: what was done, what was tested, issues encountered.

### Universal

- **session-logging** ‚Äî Records what was done, decisions made, and open items at end of session. Compatible with all agents.
- **security-audit** ‚Äî Full security audit: secrets scan, dependency audit, SAST, config review. Available to tech-lead, security, and devops agents.
